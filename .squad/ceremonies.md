# Team Ceremonies

**Status:** Initialized — ceremonies can be scheduled as needed

## Pre-Work Ceremonies

Triggered automatically before certain task types:

### Architecture Review (before major design)
- **When:** Any task labeled "architecture" or touching 3+ domains
- **Participants:** Bruce (Facilitator), all specialists (30 min)
- **Output:** Architecture decision recorded in decisions.md

### Kickoff (before project phases)
- **When:** Start of new phase (Phase 2, Phase 3, etc.)
- **Participants:** Bruce (Facilitator), all specialists (45 min)
- **Output:** Phase goals, dependencies, timeline

## Post-Work Ceremonies

Triggered after agent work completes:

### Code Review (after major work)
- **When:** PRs, major modules, security-critical code
- **Participants:** Bruce (Reviewer), author's team (async)
- **Output:** Approval or rejection w/ feedback

## On-Demand Ceremonies

Scheduled by user request:

- **Retrospective** — review progress, adjust approach
- **Planning** — decompose incoming work
- **Emergency Standup** — unblock critical issues
